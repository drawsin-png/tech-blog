최근 교통사고의 주요 원인 중 하나인 **졸음운전**을 방지하기 위한 시스템을 개발해 보았습니다. 이 글에서는 제가 며칠간 씨름하며 구축한 졸음 감지 AI 모델의 파이프라인과 트러블슈팅 경험을 공유하고자 합니다. 다른 많은 기술 블로그에서도 다루는 주제이지만, 실제 데이터를 수집하고 정제하는 과정에서의 날것의 경험을 담아보았습니다.

## 1. 프로젝트 배경 및 목표
운전자의 얼굴 상태(눈 깜빡임, 입 벌림 등)를 실시간으로 스트리밍하여 정상 상태인지, 졸음 상태인지를 분류하는 모델을 구축하는 것이 목표였습니다.

## 2. 데이터 준비와 라벨링 지옥
AI 모델의 성능은 결국 데이터가 결정한다는 유명한 문구를 이번 프로젝트에서 다시 뼈저리게 느꼈습니다.
비디오 데이터를 수집하고 'normal(정상)' 상태와 'drowsy(졸음)' 상태로 분류했습니다. 

각 클래스 당 최소 200개 이상의 고품질 샘플을 확보하는 데 집중했습니다. 이 과정에서:
- 화질이 너무 떨어지는 프레임 제거
- 측면 얼굴이나 완전히 가려진 프레임 등 이상치 제거
- 정제가 제대로 되지 않은 노이즈 데이터들을 수작업으로 Cleaning 하는 과정이 가장 오랜 시간이 걸렸습니다.

## 3. 특징 추출 (Feature Engineering)
단순 원본 이미지를 딥러닝에 태워도 되지만, 연산량을 줄이고 직관적인 패턴을 학습시키기 위해 `dlib`의 facial landmark detection을 활용했습니다. 
사람의 얼굴에서 68개의 랜드마크를 추출한 후 두 가지 주요 Feature를 도출했습니다.

1. **EAR (Eye Aspect Ratio)**: 눈이 떠있는지 감겼는지 확인하는 비율
2. **MAR (Mouth Aspect Ratio)**: 하품 여부를 확인하기 위해 입이 벌어진 정도를 측정하는 비율

> 팁: EAR과 MAR 공식을 함수화하여 각 프레임마다 계산된 Float 값을 시계열 데이터 혹은 테이블 데이터처럼 쌓았습니다.

## 4. 모델 학습 (Support Vector Machine)
추출한 EAR, MAR 특징 값을 기반으로 가장 직관적이고 가벼운 **SVM (Support Vector Machine)** 모델을 선택했습니다.
Scikit-Learn 라이브러리를 활용해 학습을 진행했고, RBF 커널을 사용하여 비선형적인 패턴(눈은 감겨있는데 하품은 안 하는 경우 등)도 잘 파악할 수 있도록 모델을 튜닝했습니다.

## 5. 실시간 모니터링 시스템 구축
마지막으로 웹캠을 연동하여 실시간으로 모델이 제대로 작동하는지 검증하는(Real-time Monitoring) 스크립트를 작성했습니다. OpenCV 프레임을 읽고 -> dlib 랜드마크 -> EAR, MAR 계산 -> SVM 예측 -> 화면에 알림(Alert) 표시의 파이프라인입니다.

## 🚀 회고 및 느낀 점
- **데이터가 깡패다**: 모델의 정확도가 안 나올 때 처음에는 파라미터 튜닝만 하려 했는데, 결국 정답은 노이즈 데이터를 걸러내는 것이었습니다.
- **최적화 문제**: 실시간 스트리밍에서 생각보다 리소스 소모가 컸습니다. Frame drop이 생기지 않도록 매 프레임 모델 추론을 하지 않고 frame 결괏값을 평균(Rolling average) 내는 아이디어도 고민해 봐야겠습니다.

앞으로 딥러닝(CNN) 기반 모델과 비교 분석을 해보는 것도 재밌을 것 같습니다. 읽어주셔서 감사합니다!
