유튜브 영상의 썸네일과 제목이 얼마나 효과적인지 파악하기 위해서는 **CTR(Click-Through Rate, 노출 클릭률)** 데이터가 생명과도 같습니다. YouTube Studio에서 하나씩 영상을 눌러보며 확인하는 건 너무 소모적인 일이죠. 그래서 저는 파이썬을 활용해 모든 유튜브 비디오의 제목과 CTR 값을 자동으로 수집하여 테이블(Table) 형태로 저장하는 프로젝트를 진행했습니다.

## 🎯 목표
1. 구독 중인 내 채널의 모든 비디오 목록 수집
2. 각 비디오의 최근 '노출 클릭률(CTR)' 데이터 가져오기
3. {비디오 제목 : CTR 값} 구조를 가진 깔끔한 데이터 테이블 형식으로 Output 도출

## 🚫 직면한 문제점 (Troubleshooting)

### 로그인 세션 이슈
처음에는 단순히 BeautifulSoup과 Requests를 활용하려 했으나, 크리에이터 스튜디오 데이터는 철저히 **로그인 세션(Authentication)**이 필요한 페이지 뒤에 숨어 있었습니다. 따라서 단순 정적 스크래핑으로는 불가능했습니다.

### 서버 용량 부족 (ENOSPC: No space left on device)
대규모 데이터 수집 시도를 하다 캐시 데이터와 로그 데이터가 쌓여 서버/로컬 환경에 디스크 용량이 꽉 차는 문제가 생겼습니다. 결국 리눅스 환경에서 명령어(`du -sh`, `df -h`)를 돌려 캐시 디렉터리들을 청소하는 우여곡절을 겪어야 했습니다. (의외로 숨김 폴더인 `.cache/google-chrome` 등 브라우저 관련 데이터가 엄청나더군요.)

## 🛠️ 해결 방식

- 로그인 세션을 쿠키 형태로 저장하여 재접속 시 매번 수동 로그인하는 번거로움을 피했습니다. (웹 드라이버 활용 시 프로필 연동)
- Headless Browser를 사용하지 않으면 너무 느리기 때문에 최적화를 적용했고, API 할당량과 IP 밴을 피하기 위해 적절한 `time.sleep()` 대기 시간을 추가했습니다.
- 추출한 Raw 데이터는 Pandas DataFrame을 활용해 깔끔한 CSV 파일(`.csv`)로 파싱했습니다.

## ✨ 결론
단순 반복 업무는 역시 스크립트로 자동화하는 짜릿함이 최고입니다. 이제 완성된 테이블을 보고, CTR이 저조한 영상의 **썸네일 기획**과 **문구(Copywriting) 변경**을 데이터 기반으로 시도해 볼 계획입니다! 

자동화 스크립트를 짜면서, 크롤링도 좋지만 정식 YouTube Data API v3를 활용하는 게 훨씬 우아하겠다는 교훈도 얻었습니다. 다음 프로젝트로는 API 연동 버전을 고려 중입니다.
