센서 없이 오로지 **수학(Math)**과 **시간(Time)**만을 이용해 차량이 스스로 부드럽게 뱀처럼 S자를 그리며 나아가게 만들 수 있을까요? 
이번 프로젝트(일명 '인민의 탱크')는 아두이노와 4개의 TT 기어드 모터를 활용하여 기구학과 물리 속도 제어 지식만으로 로봇을 제어해보는 흥미로운 도전이었습니다.

## 1. 하드웨어 스펙과 한계점
- **타이어 지름 ($D$)**: 약 0.066m (66mm). 아두이노 스마트카 표준 노란색 바퀴입니다.
- **윤거 ($W$) & 축거 ($L$)**: 각각 0.15m, 0.13m. 이 값을 알아야 좌우 바퀴 속도를 계산할 수 있습니다.
- **모터 및 바닥 마찰**: 기본 제공되는 저렴한 TT 감속 모터는 좌우 출력이 완전히 동일하지 않다는 단점이 있었습니다. 바닥 환경 또한 미끄러짐(스핀) 현상을 발생시켜 `SLIP_FACTOR`라는 보정 값을 시스템에 적용해야만 했습니다.

## 2. 물고기, 혹은 뱀처럼 나아가기 위한 수학 모델
로봇이 직진만 하지 않고 S자 형태를 유지하기 위해선 주기적인 회전 속도의 변화가 필요합니다. 우리는 가장 완벽한 파동 함수인 `sin()`을 도입했습니다.

### 회전 각속도 ($\omega$)
로봇이 코를 꺾는 속도는 시간에 따른 사인(Sine) 함수로 모델링했습니다.
$$\omega(t) = \Omega_{max} \times \sin(\frac{2\pi \cdot t}{T})$$
$\Omega_{max}$ 값을 키우면 로봇이 좌우로 더욱 급격하게 꺾이며 곡률 반경이 짧아집니다.

### 스키드 조향 (Skid Steering) 및 속도 분배
S자를 그리며 전진하기 위해서는 목표 전진 속도($V$)에 회전 각속도 성분을 각각 더하고 빼서 좌우측 바퀴의 속도를 다르게 만들어야 합니다.
- **좌측 속도**: $V_{Left} = V - \frac{W \cdot \omega}{2}$
- **우측 속도**: $V_{Right} = V + \frac{W \cdot \omega}{2}$

## 3. 코드 구현의 핵심: 물리 속도를 PWM으로
이런 수식들을 실제로 모터가 알아듣는 전기 신호로 번역하는 과정이 코드의 핵심입니다. m/s(초속)를 아두이노의 PWM(0~255) Duty Cycle로 바꿔주는 `mpsToPWM()` 함수를 작성했습니다.

```cpp
int mpsToPWM(float mps) {
  if (mps < 0) return 0;

  // 1. 선속도 -> 바퀴 RPM 변환 공학 공식
  float rpm = (mps * 60.0) / (PI * WHEEL_DIAMETER);

  // 2. 모터 한계점 비례식 변환
  float pwm_val = (rpm / MAX_RPM) * 255.0;

  return (int)pwm_val;
}
```

이외에도 앞바퀴(조향측)에 `FRONT_BIAS`를 곱해 힘을 더 주고, 직진성을 잃게 만드는 하드웨어 불균형을 잡기 위해 `RIGHT_TRIM` 값을 미세 조정하는 작업이 포함되어 있습니다.

## 💡 회고 및 향후 발전 가능성
싸구려 모터라 하더라도, 물리학과 기구학을 바탕으로 세밀하게 수학적인 보정을 소프트웨어에서 해주면 생각보다 우아하고 예쁜 S자 주행을 구현해낼 수 있다는 걸 깨달았습니다. 다음 목표는 여기에 라이다나 비전 카메라 센서를 탑재하여 주변을 인식하면서 장애물을 회피하는 더 똑똑한 조향 로직을 짜는 것입니다.
